## ⚡ Tổng quan Dự án

**MiroFish** là một công cụ dự đoán AI thế hệ mới dựa trên công nghệ multi-agent. Bằng cách trích xuất thông tin hạt giống từ thế giới thực (như tin tức đột phá, dự thảo chính sách, tín hiệu tài chính), tự động xây dựng thế giới kỹ thuật số song song độ trung thực cao. Trong không gian này, hàng ngàn agent thông minh có tính cách độc lập, bộ nhớ dài hạn và logic hành động tự do tương tác và phát triển xã hội. Bạn có thể quan sát từ "góc độ của Chúa" và động tiêm biến số, dự đoán chính xác hướng phát triển tương lai——**hãy cho tương lai diễn tập trên bàn cát kỹ thuật số, giúp quyết định chiến thắng sau một trăm cuộc mô phỏng**.

> Bạn chỉ cần: tải lên tài liệu hạt giống (báo cáo phân tích dữ liệu hoặc câu chuyện tiểu thuyết thú vị)</br>
> MiroFish sẽ trả lại: một báo cáo dự đoán chi tiết và một thế giới kỹ thuật số song song có tính tương tác cao

### Tầm nhìn của chúng tôi

MiroFish cam kết xây dựng gương phản ánh trí tuệ tập thể của thực tế, bằng cách nắm bắt sự nổi lên tập thể từ tương tác của các cá nhân, vượt qua giới hạn của dự đoán truyền thống:

- **Ở cấp độ vĩ mô**: Chúng tôi là phòng thí nghiệm diễn tập cho các nhà quyết định, cho phép chính sách và công cộng thử và sai trong môi trường không rủi ro
- **Ở cấp độ vi mô**: Chúng tôi là bàn cát sáng tạo cho người dùng cá nhân, cho dù đó là dự đoán kết thúc tiểu thuyết hay khám phá những ý tưởng hoang đường, tất cả đều có thể thú vị, vui nhộn và có thể tiếp cận

Từ dự đoán nghiêm túc đến mô phỏng vui nhộn, chúng tôi làm cho mọi điều có thể được nhìn thấy kết quả, làm cho dự đoán mọi thứ trở thành có thể.

## 🔄 Quy trình Làm việc

1. **Xây dựng Đồ thị**: Trích xuất Hạt giống Thực tế & Tiêm Bộ nhớ Cá nhân và Tập thể & Xây dựng GraphRAG
2. **Thiết lập Môi trường**: Trích xuất Quan hệ Thực thể & Tạo Tính cách Nhân vật & Agent Cấu hình Môi trường Tiêm Tham số Mô phỏng
3. **Bắt đầu Mô phỏng**: Mô phỏng Nền tảng Kép Song song & Tự động Phân tích Yêu cầu Dự đoán & Cập nhật Bộ nhớ Chuỗi thời gian Động
4. **Tạo Báo cáo**: ReportAgent sở hữu một bộ công cụ phong phú để tương tác sâu với môi trường sau mô phỏng
5. **Tương tác Sâu**: Trò chuyện với bất kỳ ai trong thế giới mô phỏng & Trò chuyện với ReportAgent

## 🚀 Bắt đầu Nhanh

### Một, Triển khai Mã nguồn (Được khuyến nghị)

#### Yêu cầu Tiên quyết

| Công cụ | Yêu cầu Phiên bản | Giải thích | Kiểm tra Cài đặt |
|------|---------|------|---------|
| **Node.js** | 18+ | Môi trường chạy Front-end, bao gồm npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Môi trường chạy Back-end | `python --version` |
| **uv** | Phiên bản mới nhất | Trình quản lý Gói Python | `uv --version` |

#### 1. Cấu hình Biến môi trường

```bash
# Sao chép tệp cấu hình mẫu
cp .env.example .env

# Chỉnh sửa tệp .env, điền khóa API cần thiết
```

**Biến môi trường Bắt buộc:**

```env
# Cấu hình LLM API (hỗ trợ bất kỳ LLM API nào ở định dạng OpenAI SDK)
# Khuyến nghị sử dụng mô hình qwen-plus trên nền tảng Bailian của Alibaba: https://bailian.console.aliyun.com/
# Lưu ý mức tiêu thụ lớn, có thể thử mô phỏng dưới 40 vòng trước
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Cấu hình Zep Cloud
# Hạn ngạch miễn phí hàng tháng có thể hỗ trợ việc sử dụng đơn giản: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Cài đặt Phụ thuộc

```bash
# Cài đặt tất cả phụ thuộc cùng một lúc (thư mục gốc + front-end + back-end)
npm run setup:all
```

Hoặc cài đặt từng bước:

```bash
# Cài đặt phụ thuộc Node (thư mục gốc + front-end)
npm run setup

# Cài đặt phụ thuộc Python (back-end, tự động tạo môi trường ảo)
npm run setup:backend
```

#### 3. Khởi động Dịch vụ

```bash
# Khởi động cả front-end và back-end cùng một lúc (thực thi trong thư mục gốc dự án)
npm run dev
```

**Địa chỉ Dịch vụ:**
- Front-end: `http://localhost:3000`
- Back-end API: `http://localhost:5001`

**Khởi động Riêng lẻ:**

```bash
npm run backend   # Chỉ khởi động back-end
npm run frontend  # Chỉ khởi động front-end
```

### Hai, Triển khai Docker

```bash
# 1. Cấu hình Biến môi trường (tương tự triển khai mã nguồn)
cp .env.example .env

# 2. Kéo Hình ảnh và khởi động
docker compose up -d
```

Mặc định sẽ đọc `.env` trong thư mục gốc và ánh xạ cổng `3000（front-end）/5001（back-end）`

> Đã cung cấp địa chỉ hình ảnh tăng tốc thông qua nhận xét trong `docker-compose.yml`, có thể thay thế khi cần thiết
